# 金字塔原理

## 序言的具体写法

### 序言的目的

- 让读者抛开其他思想，专注于你的话题。
- 激发读者兴趣，吸引注意力。

### 序言的形式

- 文章的序言通常会采用讲故事的形式
  - 为什么？
    - 读者只有在感受到强烈的吸引力时，才会愿意暂时放弃其他思想，专注于你提供的信息。
    - 从心理学角度看，与其让读者在混乱的思想中摸索相比，先向他们传递一些简单易懂、容易接受的信息，可以使读者更容易接受全文的思想。
    - 讲故事还会让读者觉得作者得出结论的逻辑必然是正确的，从而减少读者对文章随后的思路提出反对意见。
  - 怎么写？
    - 序言应当先介绍读者熟悉的某些"背景"，说明发生的"冲突"，并由此引发读者的"疑问"，然后针对该"疑问"给出"答案"。

### 序言的基本结构

- 4个要素（SCQA结构）
  - 介绍背景（S, Situation）
    - 引发读者思考"你为什么要告诉我这些呢？"，从而引出"冲突"
  - 指出冲突（C, Complication）
    - 指出某个问题，或者推动故事情节发展，从而引出"疑问"
  - 引发疑问（Q, Question）
  - 给出答案（A, Answer）
- 4个要素顺序
  - 标准式：背景—冲突—答案
    - 娓娓道来，自然流畅。平稳引导听众思路，易于理解和接受。常见于技术方案讲解场景
  - 开门见山式：答案—背景—冲突
    - 答案先行，高效直接。快速传递核心结论，节省时间，抓住重点。常见于线上问题处理场景
  - 突出忧虑式：冲突—背景—答案
    - 冲突先行，引发重视。强调问题严重性，激发改变意愿，争取资源。常见于风险同步场景
  - 突出信心式：疑问—背景—冲突—答案
    - 悬念先行，激发共鸣。吸引全场注意，塑造专业形象。常见于产品发布场景

### 序言的关键句要点

- 关键句要点的作用
  - 关键句要点是答案的延伸，表达文章的思想
  - 让读者在开始阅读的最初30秒内就能了解你的全部思路。随后的内容只是解释或支持这些要点
  - 让读者明白，接下来不会有令人意想不到的内容，这样他们在继续阅读时会更加容易接受和理解

### 写序言的注意事项

- 千万不要罗列名词。关键句要点应当是完整的思想，完整的句子。
  - 用这种方式列出的要点不是在向读者传达文章的信息，而只是向他们硬塞了一系列无法准确理解的词汇，除了浪费读者的时间，拖延读者对文章的理解之外，没有任何用处。

### 序言的长度要求

- 序言的长度应确保在你可以引导读者按你的思路思考，读者和你"站在同一位置"。你与读者的关系越近，序言的篇幅就可以越短。但无论有多短，都必须能唤起读者的疑问。
  - 我们在梳理技术方案或者跨团队沟通时也是一样，如果对方未接触过相关业务，也需尽可能详尽阐述。如果高频接触的，比如组内相互review的同学，梳理时也可以简短一些。

### 写好序言的基本原则

- 引导读者：序言的目的是"提示"读者而不是"告诉"读者某些信息。
- 满足基本结构：序言必须包含故事所需要的3个要素，即"背景""冲突"和"答案"。
  - 为什么不包含"疑问"？
    - 可明确写出，也可留给读者自行推导
    - 背景和冲突可推理出疑问，如果再将这个疑问写出来，反而可能显得多余。
- 长度合适：序言的长度取决于读者和主题的需要。

### 序言的常见模式

- 它们都基于SCQA（背景-冲突-疑问-答案）结构，但侧重点和适用场景不同。
- 1. 发出指示式（针对"我们应该做什么"等问句）
  - 答案侧重于具体的行动指令或操作步骤
  - 示例
    - 情境（Situation）：团队的月度例会是同步项目进展、解决关键问题的重要平台。
    - 冲突（Complication）：然而，近期会议常出现议题发散、准备不足、议而不决等情况，导致效率不高，占用了大量宝贵时间。
    - 答案（Answer）：为确保会议价值，现推行新的会议规范，具体要求如下：第一，会议组织者需提前24小时发出包含明确议题的议程；第二，每位参会者需提前阅读材料并准备简要更新；第三，会议最后10分钟必须明确后续行动项、负责人和截止日期。
  - 常用于下达任务、布置工作的场景
- 2. 请求支持式（针对"我们是否应该这样做"等问句）
  - 答案是一个需要被批准的建议或方案，核心目的是争取支持
  - 示例
    - 情境（Situation）：如上所述，我们当前的月度例会在效率和决策方面有提升空间。
    - 冲突（Complication）：为系统性地解决此问题，我草拟了一套引入专业会议引导技术和协同办公软件的计划，预计能显著改善现状，但该计划的全面实施需要申请一笔额外的预算支持。
    - 答案（Answer）：恳请您批准此项预算，因为：投资明确：能有效将会议效率提升约30%，并改善决策质量；回报清晰：节省的时间成本将直接转化为生产力。
  - 常用于申请经费、寻求批准、争取资源的场景
- 3. 解释做法式（针对"我们应该如何做"等问句）
  - 答案侧重于解释解决问题的具体步骤、方法或流程
  - 示例
    - 情境（Situation）：我们希望本次月度例会能更高效地得出结论。
    - 冲突（Complication）：过去开放的讨论模式虽然全面，但有时难以聚焦并快速形成决议。
    - 答案（Answer）：这次我们将尝试采用"结构化决策法"来聚焦议题。具体步骤如下：第一步，用5分钟明确陈述待决议题；第二步，每人1分钟陈述核心观点；第三步，集体讨论并澄清疑问；第四步，进行匿名投票以收集真实意见。
  - 常用于提供解决方案、指导操作的场景
- 4. 比较选择式（针对"我们应该做什么"等问句）
  - 答案是一个经过比较分析后得出的优选方案或决策建议
  - 示例
    - 情境（Situation）：团队一致认为需要改进月度例会的形式以提升参与度。
    - 冲突（Complication）：目前收集到几个不同方案：A方案是将会议拆分为更频繁的短会；B方案是引入新的互动工具并保持每月一次。
    - 答案（Answer）：综合分析，我推荐B方案。主要依据是：可行性：B方案对现有工作节奏改变最小，更易推行；有效性：新工具能解决互动性不足的核心痛点；成本效益：B方案的投入产出比更高。
  - 常用于方案评审、项目立项的场景

## 读书感受

- 写好序言，有助于帮助读者以整体的视角去理解文章。在日常工作、跨团队协作中也可以运用到。
- 我在以往的阅读中，比较习惯性跳过序言。看了这个章节后，明白了序言的作用，后续阅读过程中也会去看这块内容。
